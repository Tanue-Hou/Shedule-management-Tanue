# Obsidian Schedule Manager (长周期日程管理 Skill)

专为基于大语言模型的 Agent（如 Hermes）设计，深度结合 Obsidian 的原生特性，打造一套“零幻觉、零遗忘”的长周期（如 34 周大计）日程管理智能工作流。

## 💡 为什么需要这个 Skill？

当尝试让大语言模型（LLM）记住长达几十周的复杂任务安排时，往往会因为上下文长度（Context Window）的限制和时间推移，导致极度严重的**幻觉、遗忘和新旧任务混淆**。
本 Skill 提出了一种治本的全新范式：**将“记忆”与“逻辑”彻底分离。**
Agent 不再使用自己那不可靠的“大脑”硬记日程，而是通过专门编写的 Python 脚本引擎，直接读写您电脑本地的 **Obsidian 笔记**，将您的私人知识库作为“唯一真相来源 (Source of Truth)”。

## ✨ 核心特性

- **Obsidian 完美兼容**：脚本引擎专门为 Markdown 打造，天然理解 `## Week 1` 与 `- [ ]` Checklist 语法格式。
- **Block ID 像素级追踪**：系统自动为每一条任务生成并隐式绑定 Obsidian 块链接标识符（如 `^a1b2c3`）。无论您的笔记排版多长多复杂，插入了多少 `[[双向链接]]` 或 `#标签`，Agent 都能通过脚本一招锁定特定行，并在不破坏原文件任何格式的前提下精准打勾或修改！
- **自愈式架构 (Auto-healing)**：普适性拉满！就算您平时顺手在 Obsidian 某一行敲下普通任务，忘加 Block ID，脚本在下一次运行时也会自动发现它、温柔地在行尾补上隐藏 ID，立刻纳入全自动追踪版图，绝不漏跟。
- **动态规划顺延 (Rollover)**：长周期计划最怕遗留任务成烂尾楼。使用该系统，只需每周一句“整理一下任务”，系统自动切走过去所有未做的废弃清单，集中转移到本周名录下，并打上醒目的 `#顺延` 标签。
- **一键大纲注入**：可以将整段粗糙的计划文本发给 Agent，它会通过强大的批处理流一条条帮您转化为合规的任务对象并注入 Obsidian。

## 📂 文件结构

- `SKILL.md`: 喂给大模型 / Agent 吃的核心 Prompt 与“铁血规矩”。强制模型必须调用工具，严禁造假。
- `scripts/obsidian_schedule.py`: 本系统的核心“机械臂”。健壮的 Python 解析与修改引擎。

## 🚀 部署与使用向导

1. **导入**：将整个仓库克隆或放置于您的 Agent 技能库中（例如 `~/.gemini/config/skills/dynamic-schedule-manager`）。
2. **挂载**：使用任意编辑器打开 `scripts/obsidian_schedule.py`。
3. **定位**：修改第 9 行的 `OBSIDIAN_FILE` 变量，将其精确指向您的 Obsidian 库中负责日程的 Markdown 文件（例如 `/Users/yourname/Documents/ObsidianVault/34周研究计划.md`）。
4. **赋权**：如果终端未授权，执行 `chmod +x scripts/obsidian_schedule.py`。

## 🗣️ 人机交互范例

部署完成后，在日常与 Agent 聊天时，您可以随便吩咐：
- *“在第 5 周添加一个任务：撰写第二章初稿。”*
- *“我这周（第3周）有什么进度要赶的？”*
- *“把上次那个‘补充实验数据’的活儿推迟到下周去。”*
- *“进入下一周了，将之前落下的没做完的任务全部顺延过来吧。”*

Agent 将化身最忠实的打字员，让您的 Obsidian 画面实时跳动更新！
